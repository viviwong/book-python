02-gitbook发布

02-gitbook发布

1 构建书籍

gitbook build 将书籍内容输出到默认目录，也就是当前目录下的 _book 目录

$ gitbook build
Starting build ...
Successfully built!

$ ls _book
index.html       chapter0          chapter1            chapter2            gitbook       index.html          search_index.json

2 创建新仓库和gh-pages分支

  • 登录到Github，创建一个新的仓库，名称我们就命令为book-python，这样我就就得到了一个book-python的空仓库。
  • 克隆仓库到本地：git clone git@github.com:USER_NAME/book-python.git。
  • 创建一个新分支：git checkout -b gh-pages，注意，分支名必须为gh-pages`。
  • 将分支push到仓库：`git push -u origin gh-pages。
  • 切换到主分支: git checkout master。

3 同步静态网站代码到分支

将_book目录中的代码复制到gh-pages分支中去

  • 切换gh-pages分支。我们需要将gh-pages分支内容存放到另一个目录中去。
  • 克隆gh-pages分支：git clone -b gh-pages git@github.com:USERNAME/book.git book-end。这步我们只克隆了gh-pages分支，并存放在一个新的目录book-end里面。
  • copy _book目录中的代码到book-end目录中。
  • 将book-end目录push到远端仓库

4 持续更新方法

  • 正常编写gitbook,到gitbook书籍目录里面编写书籍,比如我现在书籍目录叫book-python
  • 目录结构如果有修改使用 gitbook init
  • 编译书籍 gitbook build
  • 复制_book目录里面所有文件到book-end目录中
  • 进入book-end目录
  • 执行版本控制命令 git add .添加所有文件到版本库 git commit -m '留言'做版本并且留言 git push提交到远端仓库

注意 book-python和book-end不要在一个目录下
